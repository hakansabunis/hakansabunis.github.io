Overview

This project focuses on building a complete Digital Audio Broadcasting (DAB+) transmission chain using Software Defined Radio tools and open-source broadcasting software. The system ingests encoded audio, multiplexes multiple services, performs OFDM modulation, and transmits the signal over-the-air with PlutoSDR. The implementation also included MOT Slideshow (images & text messages) and a prototype Emergency Warning System (EWS) override path based on CAP XML.

Objectives

  • Design and deploy a full DAB+ pipeline (encoding ‚Üí multiplexing ‚Üí modulation ‚Üí SDR transmission).
  • Configure ensemble/services with correct labels, PTY, ECC, and language fields.
  • Enable and test MOT Slideshow and Dynamic Label Segment (DLS).
  • Prototype an Emergency Warning System (EWS) signal override scenario.

System Design

  • Encoding: ODR-AudioEnc (HE-AAC v2, stereo audio).
  • Multiplexing: ODR-DabMux (INFO config files for ensemble, services, subchannels, MOT, PAD/DLS).
  • Modulation: ODR-DabMod (OFDM Mode I, guard interval & interleaving per ETSI spec).
  • Transmission: PlutoSDR (frequency calibration, TX gain, sampling rate adjustments).
  • Monitoring: dablin/dablin_gtk for live reception, Wireshark for EDI/ETI stream inspection.

Results

  • Achieved stable DAB+ on-air transmission with audio services successfully decoded on receivers.
  • MOT Slideshow and Dynamic Label messages displayed correctly.
  • Documented configuration templates for faster ensemble bring-up and testing.
  • EWS manual test path validated: CAP XML alert triggered priority audio/visual override.
